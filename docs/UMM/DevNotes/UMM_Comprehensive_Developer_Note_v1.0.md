# **2. Background: Why This Architecture Exists**

UMM’s earliest incidents revealed a core architectural vulnerability:

> **Documentation misclassification → subsystem confusion → multi‑plane drift.**

This drift vector caused:

- the **Dices Public Faces** misclassification  
- the **Outreach Packet** subsystem confusion  
- HRB runaway modeling  
- narrative bleed  
- traversal misbinding  
- governance contamination  

These incidents forced the creation of:

- Safeguards  
- Final Safety Net  
- SIAP Spine  
- Drift Vector Mitigation Protocol  
- Safe Hooking Protocol  
- Hook Safety CI  
- The Systems Integration Document (SID)  

This Developer Note explains how these systems evolved and why they matter.
