# **5. NEW: Auto‑Detect Rendering Layer (Mobile vs Desktop)**  
*(This is the section you requested.)*

ROOT now includes a **device‑aware rendering rule**:

> **If the user is on desktop → serve SVG.  
If the user is on mobile → serve PNG.**

This is implemented as a **governance directive**, not code, because GitHub does not allow runtime scripts in SVG.

### **5.1 Detection Logic (Governance-Level)**

ROOT instructs all NDH documentation, READMEs, and index pages:

```
If user-agent contains "Mobile" or "Android" or "iPhone":
    Display PNG fallback
Else:
    Display SVG primary
```

### **5.2 NDH-Compliant Linking Pattern**

Every NDH artifact must link like this:

```
Primary (SVG): /svg/NDH-Visual-Compendium-v1.0.svg
Fallback (PNG): /png/NDH-Visual-Compendium-v1.0.png
```

And ROOT mandates the following display rule:

```
Desktop View:
    Show SVG first, PNG second

Mobile View:
    Show PNG first, SVG second
```

### **5.3 README Auto‑Routing Section**

ROOT requires all READMEs to include:

```
### Rendering Notice
GitHub Mobile cannot render complex SVG geometry.
If this page appears blank, open the PNG fallback:
→ /png/NDH-Visual-Compendium-v1.0.png
```

This ensures **automatic human routing** based on device behavior.
