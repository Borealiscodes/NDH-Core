1. **Meta Index v1.0** — the ontology / self‑model

3. **Workflow Spine v1.0** — the procedural control loop

- rigid

- structurally correct but not yet adaptive  

SID will change that.

# ⭐ 4. Current NDH Limitations (Pre‑SID)

These are the three risks you identified — and they are real:

## 4.1 Rigidity  
NDH cannot dynamically adjust constraint strength.  
Pins are fixed.  
Overlays are fixed.  
Rendering safety is fixed.

This limits adaptation.

## 4.2 Complexity Overhead  
Meta Index + Workflow Spine + visuals = cognitive cost.  
The system is powerful but heavy.

## 4.3 Single‑Spine Dependence  
If the Workflow Spine is wrong, everything downstream inherits the error.

This is the largest structural risk.

SID is designed specifically to eliminate these.
