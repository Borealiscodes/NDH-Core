## **Overview**

The Runtime Plane is the **live execution layer** of the Unified Meta‑Model (UMM).  
It applies constitutional compassion, enforces boundaries, preserves identity, manages continuity, and coordinates epochal transitions.

This index provides a complete map of all Runtime Plane components.

## **Primary Components**

### **1. Compassion Runtime Engine**  
`/UMM/runtime/COMPASSION-RUNTIME-ENGINE_v1.0.md`  
The central gatekeeper for all user‑affecting operations.

Responsibilities:

- enforce compassion invariants  
- soften or block harmful transitions  
- maintain ND‑accessible pacing  
- coordinate with Mimi  
- dispatch validated operations to core engines  

---

### **2. Core Engines**  
`/UMM/runtime/core-engines/`

Each engine enforces one constitutional dimension.

- **Boundary Engine** — non‑coercive access control  
- **Continuity Engine** — gentle state transitions  
- **Identity Engine** — identity preservation  
- **Epoch Engine** — macro‑state and versioning  
- **Constitutional Engine** — constitutional compliance  

See: **Core Engines README**
