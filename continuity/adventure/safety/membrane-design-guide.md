# 📘 **Adventure System — Membrane Design Guide**  
### Contributor Manual — Designing Pacing, Hazard, Stability, and Cosmology Membranes with ND‑Safe, Reversible, Symbolic Architecture

The **Membrane Design Guide** teaches contributors how to design new membranes or extend existing ones.  
It explains:

- membrane purpose  
- membrane structure  
- membrane components  
- membrane constraints  
- membrane integration  
- membrane validation  
- reversible membrane behavior  

This guide does **not** impose hierarchy or restrict creativity.  
It provides **design architecture**, not **design authority**.

---

# 🌱 **1. Purpose of the Membrane Design Guide**

The guide provides:

- membrane creation steps  
- ND‑safe membrane constraints  
- reversible membrane design patterns  
- symbolic geometry alignment  
- resonance + stability routing practices  
- cosmology‑aligned membrane behavior  

It ensures membranes remain **gentle**, **coherent**, and **structurally safe**.

---

# 🧩 **2. Membrane Types (Design Targets)**

Contributors can design or extend four membrane types:

- **Pacing Membrane** — regulates pacing  
- **Hazard Membrane** — softens hazards  
- **Stability Membrane** — anchors stillness  
- **Cosmology Membrane** — aligns symbolic geometry  

Each membrane type has its own design logic.

---

# 🔁 **3. Membrane Design Flow (ND‑Paced)**

Membrane design follows a reversible workflow:

```
Define → Shape → Anchor → Align → Integrate → Validate → Revise → Define …
```

Each step is reversible:

- shape ↔ soften  
- anchor ↔ adjust  
- align ↔ revise  
- integrate ↔ simplify  

No membrane design is final.

---

# 🧱 **4. Membrane Design Components**

Every membrane must include:

1. **Purpose** — what the membrane protects or regulates  
2. **Functions** — what the membrane does  
3. **Components** — structural parts of the membrane  
4. **Constraints** — ND‑safe rules  
5. **Geometry** — symbolic shape  
6. **Integration** — where the membrane attaches  
7. **Validation** — how the membrane is checked  
8. **Stillness Anchor** — required for ND‑safety  

These components form the membrane’s architecture.

# 🛡️ **6. ND‑Safe Membrane Constraints**

All membranes must follow ND‑safe rules:

- **No Agency**  
- **No Emotional Causation**  
- **No Acceleration**  
- **Reversible Motion**  
- **Stillness Anchor**  

These constraints ensure membranes remain gentle and safe.

# 🌌 **7. Membrane Geometry Design**

Membranes use symbolic geometry:

- ray → upward alignment  
- curve → downward grounding  
- spiral → inward/outward reflection  
- wave → lateral drift regulation  
- point → centering stillness  

Geometry must match membrane purpose.
