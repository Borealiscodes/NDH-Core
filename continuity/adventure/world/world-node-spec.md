# 📘 **Adventure System — World Node Specification**  
### Continuity Architecture — World-Level Structural, Affective, and Cosmological Behavior

A **World Node** is the foundational environment unit of the Adventure System.  
It defines the **space**, **climate**, **structure**, **resonance**, and **membrane behavior** of a world.  
World Nodes do not imply agency, emotion, or narrative intent — they provide **environmental architecture**.

---

# 🌍 **1. Purpose of a World Node**

A World Node provides:

- spatial structure  
- environmental climate  
- resonance alignment  
- stability membrane anchoring  
- pacing-safe environmental transitions  
- symbolic cosmology alignment  
- reversible environmental motion  

World Nodes are **non-sentient**, **non-agentic**, and **ND-safe**.

They define **where** the adventure happens, not **what** happens.

---

# 🧱 **2. World Node Components**

A World Node consists of six structural components.

## **2.1 Spatial Frame**
Defines the world’s physical or symbolic layout.

Includes:

- topology  
- boundaries  
- traversal paths  
- anchor points  
- environmental layers  

Spatial Frames do not imply realism; they define **navigable structure**.

---

## **2.2 Environmental Climate**
Defines the world’s affective atmosphere.

World Climates include:

- mosslight (glow field)  
- fog (diffusion field)  
- pulse meadow (rhythmic grid)  
- echo cavern (resonance chamber)  
- stillness grove (anchor basin)  

Each climate is reversible and ND-paced.

---

## **2.3 Resonance Profile**
Defines how the world interacts with resonance flows.

A World Node has a **six-vector resonance profile**:

- upward (clarity)  
- downward (calm)  
- inward (reflection)  
- outward (warmth)  
- lateral (drift)  
- centering (stillness)  

The profile determines how resonance behaves inside the world.

---

## **2.4 Stability Membrane**
Defines the world’s grounding and safety behavior.

Stability Membrane functions:

- centering  
- grounding  
- softening  
- reversible pacing  
- hazard neutralization  

Every world must include a **stillness anchor**.

---

## **2.5 Cosmology Alignment Layer**
Defines how the world fits into the Adventure System’s cosmology.

Includes:

- symbolic alignment  
- resonance mapping  
- environmental coloration  
- narrative compatibility  
- loop integration  

Cosmology alignment ensures worlds remain coherent across transitions.

---

## **2.6 Node-Level Metadata**
Defines world-level metadata for repo and system integration.

Metadata includes:

- world name  
- world type  
- climate type  
- resonance profile  
- stability membrane type  
- cosmology alignment tag  
- integration notes  

---

# 🌫️ **3. World Node Behavior Rules**

World Nodes follow strict ND-safe constraints.

## **3.1 No Agency**
World Nodes cannot:

- want  
- intend  
- persuade  
- direct  

They are structural, not agentic.

---

## **3.2 No Emotional Causation**
World Nodes cannot cause or imply emotion.

They shape **environmental climate**, not **emotional state**.

---

## **3.3 Reversible Motion**
All world-level motion must be reversible:

- brightening ↔ softening  
- opening ↔ deepening  
- drifting ↔ centering  
- resonance ↔ grounding  

No world state is final.

---

## **3.4 No Acceleration**
World Nodes cannot increase:

- urgency  
- intensity  
- emotional pressure  

All environmental motion is gentle.

---

## **3.5 Stillness Anchor Required**
Every World Node must include a **stillness anchor** — a point where environmental motion can settle safely.

---

# 🌌 **4. World Node Geometry**

World Nodes use symbolic geometry:

```
Spatial Frame → topology
Climate → field
Resonance → vector set
Stability → membrane
Cosmology → alignment layer
Metadata → structural tags
```

These geometries define **structure**, not **story**.

---

# 🔁 **5. World Node Integration Across System Layers**

## **5.1 Character Nodes**
Characters inherit environmental climate and resonance tint from the world.

## **5.2 Story Nodes**
Stories inherit pacing constraints and stability anchors from the world.

## **5.3 Engines**
Engines modulate world behavior:

- Identity Engine → resonance tint  
- Narrative Engine → pacing-safe transitions  
- Resonance Engine → coloration  
- Stability Engine → grounding  

## **5.4 Membranes**
Membranes constrain world behavior:

- Pacing Membrane → softening  
- Hazard Membrane → grounding  
- Stability Membrane → centering  
- Cosmology Membrane → symbolic alignment  

## **5.5 Loop**
World Nodes participate in the Adventure Loop:

```
World → Character → Story → World
```

Each transition carries resonance coloration and climate modulation.
