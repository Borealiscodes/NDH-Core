# 📄 **Random Adventure Generation — Reference Document**  
### *How Story Elements Become Full Adventures in the Continuity System*

---

## 1. Purpose  
This document explains how the Continuity Adventure System can take **user‑provided story elements** and automatically generate a **full multi‑mode adventure**.

It is designed for:

- contributors  
- designers  
- world‑builders  
- narrative architects  
- future expansions  

This system is **ND‑paced**, **reversible**, **gentle**, and **continuity‑safe**.

---

## 2. Core Principle  
Random adventure generation works by combining three engines:

- **World‑Builder Mode** — creates the place  
- **Character‑Creator Mode** — creates the someone  
- **Story‑Weaver Mode** — creates the moment  

The generator assembles these into a loop:

```
World → Character → Story → World → Character → Story → …
```

This loop is reversible and non‑coercive.

## 4. What the System Generates  
From the provided elements, the system automatically creates:

### 1. **World Node**  
A structured environment based on the fragments.

### 2. **Character Node**  
A reversible, ND‑paced character profile.

### 3. **Story Node**  
A soft, reversible narrative moment.

### 4. **Adventure Loop**  
A coherent sequence of:

```
World → Character → Story
```

### 5. **Chapter Progression**  
Four reversible chapters.

### 6. **Optional Systems**  
- Echo Tokens  
- Resonance Map  
- Stillness Sanctuaries  

All generated from the same fragments.
