# 📄 **Adventure Contributor Guide**  
### Continuity Adventure System — Contributor Handbook  
A calm, structured guide for contributors creating adventures in the Continuity Repo.

---

## 🌲 1. Purpose  
This guide explains how contributors can safely and gently create adventures using the Continuity Adventure System.

It covers:

- how adventures are structured  
- how to write world, character, and story elements  
- how to use the Adventure Generator  
- how to maintain ND‑paced, reversible design  
- how to keep contributions continuity‑safe  

This guide is gentle, supportive, and non‑coercive.

---

## 🧭 2. Adventure Architecture Overview  
All adventures in the Continuity System are built from **three core modes**:

- **World‑Builder Mode** — creates the place  
- **Character‑Creator Mode** — creates the someone  
- **Story‑Weaver Mode** — creates the moment  

These modes combine into the **multi‑mode adventure loop**:

```
World → Character → Story → World → Character → Story → …
```

This loop is reversible, ND‑paced, and gentle.

---

## 🌿 3. Contributor Responsibilities  
Contributors are invited to:

- write world nodes  
- create character nodes  
- design story nodes  
- add symbolic artifacts  
- expand adventure maps  
- write reversible scenes  
- maintain ND‑safe pacing  
- preserve clarity and warmth  

There are **no performance responsibilities**.  
No deadlines.  
No pressure.

## 🌳 4. Writing World Nodes  
World Nodes define the environment.

### Structure  
```
World Node:
- name
- biome elements
- stability membrane
- symbolic hazards (non-harmful)
- stillness nodes
```

### Example  
```
World Node: Mosslight Clearing
- bioluminescent echo-moss
- reversible fog layer
- resonance trees
- stillness node at center
```

## 🌫️ 5. Writing Character Nodes  
Character Nodes define the entity moving through the world.

### Structure  
```
Character Node:
- name
- intent
- tone
- ability (symbolic)
- ecology link
- reversible traits
- safety header
```

### Example  
```
Character Node: The Listener
- intent: follow the softest sound
- tone: gentle, observant
- ability: hear future echoes
- ecology link: mosslight resonance
- reversible trait: hesitant ↔ adventurous
```

## 🌌 7. Using the Adventure Generator  
Contributors can provide **any story fragments**, and the generator will assemble them into a full adventure.

### Accepted Inputs  
- world fragments  
- character fragments  
- story fragments  
- mood fragments  
- symbolic fragments  

### Example Input  
```
"a soft hum behind a tree"
"a wanderer who hesitates before stepping forward"
"a pool that reflects sound instead of light"
```

### Example Output  
- World Node  
- Character Node  
- Story Node  
- 4 reversible chapters  
- optional systems

## 🧩 8. ND‑Pacing & Reversibility  
All contributions must follow ND‑safe pacing:

- slow  
- gentle  
- breathable  
- reversible  
- non‑coercive  

Scenes must always allow:

- pausing  
- rewinding  
- soft exits  
- gentle re‑entry  

No collapse engines.  
No forced outcomes.
