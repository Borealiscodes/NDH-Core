# 📄 **Adventure Mode Creation Guide**  
### Continuity Adventure System — How to Create New Adventure Modes  
A gentle, structured guide for contributors designing entirely new adventure modes.

---

## 🌲 1. Purpose  
This guide teaches contributors how to design **new adventure modes** that integrate seamlessly with the Continuity Adventure System.

Adventure modes are:

- symbolic  
- reversible  
- ND‑paced  
- gentle  
- non‑coercive  
- continuity‑safe  

They define *how* a player interacts with a world.

---

## 🌿 2. What Is an Adventure Mode?  
An adventure mode is a **plane of interaction**.

Existing modes include:

- **World‑Builder Mode** — make a place  
- **Character‑Creator Mode** — make a someone  
- **Story‑Weaver Mode** — make a moment  
- **Realm‑Walker Mode** — explore without altering  
- **Echo‑Weaver Mode** — create symbolic fragments  
- **Dream‑Cartographer Mode** — map surreal spaces  
- **Continuity‑Stitcher Mode** — connect fragments  

Each mode defines:

- what the player does  
- how pacing works  
- how reversibility works  
- how symbolic systems behave  

---

## 🌳 3. Core Requirements for Any New Mode  
Every new adventure mode must follow these principles:

### **ND‑Pacing**
- slow  
- gentle  
- breathable  
- non‑coercive  

### **Reversibility**
- every action can be undone  
- no irreversible states  
- no collapse engines  

### **Symbolic Hazards Only**
- soft déjà vu  
- gentle disorientation  
- fading echoes  
- shifting fog  

### **Continuity Safety**
- no metaphysical triggers  
- no forced outcomes  
- no destabilizing logic  

Guided Link: **Continuity Safety**

## 🌌 4. Mode Creation Template  
Use this template to define a new adventure mode.

```
Adventure Mode: [Name]

Purpose:
- [What the mode is for]

Player Actions:
- [What the player does in this mode]

Core Mechanics:
- [Symbolic mechanics]
- [Reversible behaviors]
- [ND-paced constraints]

Pacing:
- [How slow, gentle, breathable the mode feels]

Reversibility:
- [How actions rewind or reset]

Symbolic Systems:
- [Echo Tokens, Resonance Maps, Dream Nodes, etc.]

Safety Header:
- ND-paced
- reversible
- non-coercive
- continuity-safe
```

## 🧭 7. How to Integrate a New Mode  
Every new mode must integrate with the multi‑mode loop:

```
World → Character → Story
```

Your new mode can:

- precede the loop  
- follow the loop  
- weave between nodes  
- create optional layers  
- add symbolic systems  

Examples:

- Realm‑Walker before World Node  
- Dream‑Cartographer after Story Node  
- Echo‑Weaver between Character and Story
